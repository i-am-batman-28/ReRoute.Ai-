SLIDE 2 — PROPOSED SOLUTION
================================================================================

TripGuard AI — "Your trip breaks. We fix it."

Proposed Solution:
  • AI agent that monitors trips 24/7 and auto-resolves disruptions
  • Predicts delays before official — alerts early, Plan B pre-computed
  • Multi-modal rebooking: flights + trains + combos (not flight-only)
  • Cascade engine: rebooks flight + updates hotel + flags meeting conflicts
  • Auto-files compensation claims with expense tracking (DGCA/EU261)
  • Group Sync: families/teams rebooked together on same alternative
  • Voice mode: resolve disruptions hands-free when stressed
  • Airport Survival Kit: lounge, food, charging, rest areas while waiting

How it solves the problem:
  • Zero manual effort — agent searches, ranks, and acts autonomously
  • One tap/voice to confirm → full itinerary updated
  • Whole trip fixed (not just one flight) — hotel, meetings, next legs

What makes it unique:
  • Pre-computed Plan B + Disruption Risk Score before travel
  • Full cascade handling + multi-modal + group sync + compensation
  • Real agentic AI (LLM + tools + autonomous loop), not a chatbot


SLIDE 3 — TECHNICAL APPROACH
================================================================================

Tech Stack:
  • Frontend: React (Next.js) + Tailwind CSS
  • Backend: Node.js / Python (FastAPI)
  • AI Agent: OpenAI GPT + function calling (LangChain)
  • Voice: Web Speech API
  • APIs: AviationStack (status), Amadeus (search), OpenWeatherMap,
    Google Maps (multi-modal)
  • DB: PostgreSQL / Firebase | Real-time: Socket.io
  • Notifications: FCM + Twilio (SMS/WhatsApp)

Agent Flow: [show as diagram]
  Trip Import → Risk Score + Plan B
       ↓
  Background Monitor (polls every 5–10 min)
       ↓
  Disruption Detected → Agent: Observe → Think → Act (call tools)
       ↓
  Options ranked + cascade prepared → Notify user
       ↓
  User confirms (tap/voice) → Apply: rebook + hotel + meeting + claim


SLIDE 4 — FEASIBILITY AND VIABILITY
================================================================================

Feasibility:
  • Core flow buildable in 60h: import → monitor → detect → rebook → cascade → claim
  • Flight APIs on free/sandbox tiers (AviationStack, Amadeus)
  • LLM function calling is standard — no custom training
  • Voice via browser API — no extra infra
  • Demo with mock/sandbox data — no real payments needed

Challenges & Mitigation:
  • API limits → mock/cached fallback for demo
  • Real rebooking needs airline partnerships → simulated for hackathon
  • Compensation rules vary → implement DGCA (India) for demo; extend later
  • Voice noise → tap-based primary; voice as bonus differentiator
  • Pre-recorded video backup if live demo fails


SLIDE 5 — IMPACT AND BENEFITS
================================================================================

Impact:
  • Travellers: zero effort, one-tap resolution, money recovered
  • Families: no one split up during rebooking
  • Airlines/OTAs: fewer support calls; white-label as premium feature
  • Scale: millions of disrupted trips/year; agent runs 24/7

Benefits:
  • Social: reduces stress & confusion; fair compensation for all
  • Economic: saves time + recovers money; cuts airline support cost
  • Environmental: multi-modal options reduce unnecessary flights
  • Accessibility: voice mode for elderly, stressed, less tech-savvy

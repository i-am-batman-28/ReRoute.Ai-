SLIDE 1: PROBLEM & SOLUTION
================================================================================

PROBLEM:
  Every year, 900M+ flight passengers face disruptions — delays, cancellations,
  diversions. When it happens:
    - Travelers spend 2-4 hours manually searching alternatives
    - 68% miss rebooking windows because they're asleep or unaware
    - Downstream cascade (hotels, meetings, connections) goes unmanaged
    - Compensation worth €250-€600 (EU261) goes unclaimed in 85% of cases

  The traveler is expected to figure it all out themselves, under stress,
  often at 2 AM in a foreign airport.

SOLUTION — ReRoute.AI:
  An always-on autonomous AI agent that:
    1. DETECTS disruptions in real-time (flight status + weather monitoring)
    2. FINDS alternatives automatically (searches airline inventory via Duffel NDC)
    3. REBOOKS the best option (based on user preferences — price, timing, stops)
    4. UPDATES the itinerary (cascade analysis for hotels, meetings, connections)
    5. NOTIFIES the traveler (email + in-app + real-time WebSocket push)

  Two modes:
    - AUTO-REBOOK: Agent books the best option autonomously. User wakes up rebooked.
    - MANUAL: Agent finds options, emails them with one-click confirm links.

  The traveler does nothing. The AI handles everything.

PREPROCESSING:
  - Flight status: Normalized to {delayed|cancelled|diverted|unknown} with
    delay_minutes extracted from scheduled vs estimated timestamps
  - Weather: WMO codes mapped to severity buckets (clear/light/moderate/severe)
    to prevent false alerts from minor temperature changes
  - Offers: Deduplicated by itinerary shape (route + departure window) — Duffel
    returns multiple fare classes per flight; we keep cheapest per unique journey
  - Passengers: Age classification (adult/child/infant) from DOB for Duffel
    passenger type mapping; E.164 phone normalization
  - IATA codes: 94 airports + 57 airlines with coordinates and logo URLs for
    instant resolution without external geocoding calls

AUTONOMOUS FLOW:
  Celery Beat (10 min) → Monitor scans all trips → Disruption detected?
    → YES (cancel/divert/delay≥60min) → Enqueue autonomous_disruption_handler
      → Run propose_for_trip() → Check user.auto_rebook preference
        → AUTO: confirm_and_apply(best_option) → Email "We rebooked you!"
        → MANUAL: Email with 3 options + one-click confirm links

DATA FLOW:
  Trip Snapshot (JSON) → Agent Graph → Tool Calls → Ranked Options
    → Proposal (DB) → Confirm Gate → Duffel Order → Itinerary Update
